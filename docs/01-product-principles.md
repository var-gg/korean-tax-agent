# Product Principles

1. **Agent-native first**
   - Design for users who can run AGENT AI workflows, not for a generic consumer SaaS UI first.

2. **Human approval at critical steps**
   - Authentication, high-risk classifications, and final submission must remain visible and reviewable.

3. **Public docs, private taxpayer data**
   - Architecture and workflow should be open; real taxpayer records should remain local/private.

4. **Explainable automation**
   - Automated classifications must produce reasons, confidence, and review status.

5. **Minimize consent friction, not user control**
   - Reduce the number of approval checkpoints while preserving informed consent.

6. **Docs-first implementation**
   - Product definition, scope, data model, and risk boundaries should be written before broad implementation.

7. **Composable system design**
   - Core logic, MCP interface, browser assistance, and skill instructions should evolve independently.
